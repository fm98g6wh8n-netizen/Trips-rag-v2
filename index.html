TradeGuide v2가 출처 기반의 검증된 답변을 제공하는 방법

RAG (Retrieval-Augmented Generation) 구조

TradeGuide v2는 답변을 지어내지 않습니다. 내장된 법규 문서를 먼저 검색(Retrieval)한 뒤, 검색 결과만을 근거로 답변을 구성합니다. 모든 답변에는 출처 조항과 TF-IDF 검색 점수가 함께 표시되어, 사용자가 신뢰도를 직접 확인할 수 있습니다.

2차 버전의 핵심 — Agentic + 자체 검증

선택 기능 — LLM 생성 답변 모드

기본 답변은 룰 기반 템플릿이지만, 헤더의 설정(톱니바퀴)에서 사용자 본인의 Anthropic API 키를 입력하면 검색된 법규 원문을 근거로 LLM이 답변을 생성하는 모드를 켤 수 있습니다. 이 경우에도 검색(Retrieval)은 동일한 TF-IDF 엔진이 수행하며, 생성된 답변은 같은 3유형 검증 레이어를 거칩니다 — LLM이 환각을 일으키면 신뢰도 ★★ 경고로 즉시 드러납니다. API 키는 서버가 아닌 사용자 브라우저에만 저장되며, 키가 없으면 룰 기반 모드로 동작합니다.

시스템 한계 안내. 본 시스템은 학생부종합전형 동아리 활동 산출물로, TF-IDF 기반 정적 검색과 룰 기반 답변 템플릿으로 구성됩니다. 기본 모드는 LLM 없이 동작하며, 설정에서 사용자 API 키를 입력하면 선택적으로 LLM 생성 답변을 쓸 수 있습니다(이 경우에도 동일한 3유형 검증을 거칩니다). 답변은 내장된 WTO TRIPS 협정 조항과 한미 통상법 항목 범위에서만 이뤄집니다. 검증 레이어의 "신뢰도 ★★★"은 3개 휴리스틱 규칙 통과를 의미할 뿐 법적 정확성을 보증하지 않습니다. 관세 시나리오 계산기는 단순 산술 모델이며 실제 거래에는 운임·부가세·원산지 규정 등이 추가로 반영되어야 합니다. 실제 법률·통상 판단에는 원문 및 전문가 자문이 필요합니다.
AI 답변 설정
API 키는 이 브라우저(localStorage)에만 저장되며 별도 서버로 전송되지 않습니다. 질문 시 키는 Anthropic API로만 직접 전송됩니다. 공용 기기에서는 사용 후 '키 삭제'를 눌러주세요. 키가 없거나 AI 모드가 꺼져 있으면 기존 TF-IDF 룰 기반 답변이 사용됩니다.
